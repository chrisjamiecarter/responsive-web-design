Introduction

C# is the most popular language for the .NET platform, a free, cross-platform, open-source development environment. It enables developers to write applications for IoT devices, the cloud, phones, desktops, laptops, and servers.

As a cross-platform, general-purpose language, C# boosts developer productivity while ensuring high performance. Its broad ecosystem support spans all .NET workloads. Rooted in object-oriented principles, C# also incorporates features from other paradigms, including functional programming. Low-level features allow for high-efficiency scenarios without requiring unsafe code. The majority of the .NET runtime and libraries are written in C#, ensuring that advancements in C# benefit all .NET developers.

Types

C# types are categorized into two main groups:

Value Types

Variables of value types contain the actual instance of the type.

When assigned, passed as arguments, or returned from methods, the value is copied.

Reference Types

Variables of reference types contain a reference (pointer) to the instance, not the instance itself.

When assigned, passed as arguments, or returned from methods, only the reference is copied, not the instance.

In summary, value types store the data directly, while reference types store a pointer to the data. Value types are copied by value, and reference types are copied by reference.

Keywords

Keywords are reserved identifiers with special meanings to the compiler. They cannot be used as variable or method names unless prefixed with @ (e.g., @if is valid, but if is not). There are two types of keywords in C#:

Reserved Keywords: These cannot be used as identifiers anywhere in the program. Contextual Keywords: These have special meaning only in specific contexts and can be used as identifiers outside those contexts.

As new keywords are introduced in C#, they are typically added as contextual keywords to avoid breaking existing programs written in earlier versions.

Operators and expressions

C# provides various operators that can be used with built-in types to perform basic operations. These include:

  • Arithmetic Operators: Perform arithmetic operations on numeric operands (e.g., +, -, *, /, %).
  • Comparison Operators: Compare numeric operands (e.g., ==, !=, >, <, >=, <=).
  • Logical Operators: Perform logical operations on bool operands (e.g., &&, ||, !).
  • Bitwise and Shift Operators: Perform bitwise or shift operations on integral types.
  • Equality Operators: Check if operands are equal or not (e.g., ==, !=).

These operators can be overloaded to specify behavior for user-defined types. Expressions in C# can be formed by combining literals, variables, and operators. Operator precedence and associativity determine the order of evaluation, which can be altered using parentheses.

Statements

Declaration statements

A declaration statement declares a new local variable, local constant, or local reference variable. To declare a local variable, specify its type and provide its name. You can declare multiple variables of the same type in one statement.

string greeting; int a, b, c; List<double> xs;

Exception-handling statements

Use the try-catch statement to handle exceptions that might occur during execution of a code block. Place the code where an exception might occur inside a try block. Use a catch clause to specify the base type of exceptions you want to handle in the corresponding catch block

try { var result = Process(-3, 4); Console.WriteLine($"Processing succeeded: {result}"); } catch (ArgumentException e) { Console.WriteLine($"Processing failed: {e.Message}"); }

Iteration statements

C# provides several iteration statements to repeatedly execute a block of code:

  • for loop: Executes a block of code while a specified Boolean expression is true.
  • foreach loop: Enumerates elements in a collection and executes a block of code for each element.
  • do...while loop: Executes a block of code at least once, then continues executing it while a specified condition is true.
  • while loop: Executes a block of code repeatedly while a specified condition is true. May not execute at all if the condition is initially false.

Within any of these loops, you can:

  • Use the break statement to exit the loop prematurely.
  • Use the continue statement to skip the current iteration and move to the next one.

These iteration statements allow you to control the flow of execution based on specific conditions or collections.

for (int i = 0; i < 3; i++) { Console.Write(i); } // Output: // 012

Selection statements

C# provides several conditional statements to control the flow of execution based on certain conditions or expressions:

  • if statement: Executes a block of code if a specified Boolean expression evaluates to true.
  • if-else statement: Executes one block of code if a Boolean expression is true, and another block if it's false.
  • switch statement: Evaluates an expression and executes the associated block of code based on a pattern match with the expression's value.

These statements allow you to create branching logic in your program, enabling different code paths to be executed based on specific conditions or values. The if and if-else statements are useful for simple conditional checks, while the switch statement is more suitable for multiple conditions or cases.

DisplayWeatherReport(15.0); // Output: Cold. DisplayWeatherReport(24.0); // Output: Perfect! void DisplayWeatherReport(double tempInCelsius) { if (tempInCelsius < 20.0) { Console.WriteLine("Cold."); } else { Console.WriteLine("Perfect!"); } }
